# NoteUp

> 지연 없는 필기감과 정돈된 노트 관리를 목표로 하는 태블릿 전용 필기 워크스페이스

NoteUp은 태블릿 가로 화면에서 실제 노트를 펼치듯 바로 기록하고, 작성한 노트를 노트북별로 보관할 수 있는 Android 앱입니다. 복잡한 설정 대신 펜·형광펜·지우개 같은 핵심 도구와 로컬 저장에 집중합니다.

## 핵심 가치

- **즉각적인 필기 반응**: 스타일러스 입력을 캔버스에 바로 렌더링하고, 저장 작업은 별도로 처리합니다.
- **아날로그에 가까운 사용성**: 필요한 필기 도구와 최소한의 툴바로 빠르게 기록합니다.
- **정돈된 보관**: 노트북, 노트, 페이지 구조로 작성물을 로컬에 안전하게 저장합니다.

## 주요 기능

### 노트 보관함

- 모든 노트, 미분류 노트, 노트북별 노트 필터링
- 노트북 및 노트 생성·이름 변경·삭제
- 노트 이동 및 미분류 관리
- 노트북 삭제 시 포함된 노트를 미분류로 보존
- 최근 수정일 기준 노트 카드 정렬 및 빈 상태 화면

### 필기 캔버스

- 스타일러스 전용 입력과 손바닥·손가락 입력 제외
- 압력에 반응하는 펜: 검정·파랑·빨강·초록, 3단계 굵기
- 반투명 형광펜: 노랑·초록·분홍·파랑, 3단계 굵기
- 획 단위 지우개 및 영역 지우개
- 영역 지우개 결과 실시간 미리보기
- 두 손가락 캔버스 확대·이동
- 백지·줄노트·모눈 속지와 여러 페이지 탐색·추가
- 페이지 썸네일 목록, 삭제 및 드래그 순서 변경
- 대시보드 실제 필기 썸네일
- 현재 페이지 편집 세션의 Undo / Redo
- 획을 정규화 좌표로 저장하여 화면 크기가 달라도 복원 가능
- 저장 대기 획과 저장 완료 획을 자연스럽게 전환해 필기 깜빡임 최소화

### 로컬 저장

- Room 기반 오프라인 저장
- `노트북 > 노트 > 페이지 > 획` 구조
- 새 노트 생성 시 첫 번째 빈 페이지 자동 생성
- 획 데이터는 페이지별로 저장되며, 노트·페이지 삭제 시 관계 데이터도 함께 정리
- 필기 도구와 색상·굵기·지우개 모드는 앱 재실행 후에도 복원

## 화면 구성

| 화면 | 설명 |
| --- | --- |
| 대시보드 | 좌측 사이드바에서 노트북을 선택하고, 우측 카드 목록에서 노트를 관리합니다. |
| 필기 캔버스 | 상단 툴바로 도구를 선택하고 중앙 캔버스에 스타일러스로 기록합니다. |
| 설정 | 앱 설정을 위한 기본 진입 화면입니다. |

## 기술 구성

- Kotlin / XML View / ViewBinding
- Single Activity + Fragment + Navigation Component
- MVVM: ViewModel, StateFlow, SavedStateHandle
- Room 2.8.4 + KSP 2.3.9
- Material 3, RecyclerView
- 최소 지원 Android API 26

## 프로젝트 구조

```text
app/src/main/java/com/kotlinsun/noteup
├── data        # Room, DAO, Entity, Repository, 환경설정 저장소
├── domain      # 도메인 모델과 Repository 인터페이스
└── ui
    ├── canvas      # 필기 캔버스와 툴바
    ├── dashboard   # 노트 보관함과 노트북 관리
    └── settings    # 설정 화면
```

## 실행 환경

- 태블릿 전용: smallest width 600dp 이상
- 가로 화면: `sensorLandscape`로 양방향 가로 회전 지원
- Android Studio에서 프로젝트 루트의 `NoteUp` 디렉터리를 열어 실행합니다.
